# E2E — Generate Test Artifacts

**Goal**: Produce a runnable end-to-end test script for the new surface so the developer
can execute it against staging or a local stack without reverse-engineering the API.
This block only reads source and writes to `.devloop/`, so it runs in parallel with `test`.

**Input**: Read `03-plan.md` to identify the new endpoints/commands. Read the relevant
contract files for exact request shapes.

## Step 6a — Detect the API surface

Identify what kind of interface the new functionality exposes, and pick the client tool:

| Surface | Detect via | Client in script |
|---------|-----------|------------------|
| gRPC    | `.proto` files, grpc service defs | `grpcurl` |
| REST/HTTP | `openapi.yaml`, route handlers | `curl` |
| GraphQL | `*.graphql`, resolver defs | `curl` with query body |
| CLI     | new subcommands / flags | the built binary |

If multiple apply (e.g. gRPC with HTTP transcoding), prefer the one the team tests with;
note the alternative in the report.

## Step 6b — Collect new endpoints and their shapes

From `03-plan.md`, list every new endpoint/command. For each, read the contract
(proto message, OpenAPI schema, GraphQL type, or flag set) to get exact field names,
types, and required vs optional.

## Step 6c — Determine auth requirements

Check each handler:
- Does it enforce a permission / role / scope? Note which.
- Is it open / unauthenticated? Note that too.

## Step 6d — Write the test script

Write `.devloop/<slug>/e2e-test.sh`. Order calls as a **flow** (create → read → update →
delete) so each step builds on the previous, and include the meaningful edge cases that
fell out of the implementation (idempotent repeat, not-found after delete, etc.).

```bash
#!/usr/bin/env bash
# E2E tests for <slug>
# Usage: ADDR=<addr> TOKEN=<token> bash e2e-test.sh
set -euo pipefail
ADDR="${ADDR:-<default from e2e-addr config>}"
TOKEN="${TOKEN:-}"
# test-data vars (IDs etc.) as needed

echo "==> <Step name>"
<client call using $ADDR, $TOKEN, and the request body>
```

Rules:
- One `echo "==> ..."` before each call so output is readable.
- Everything env-configurable: `ADDR`, `TOKEN`, and any test-data IDs (with `:-` defaults
  and a guard that errors if a required one is empty).
- A comment above each call states what it verifies.

## Step 6e — Write the report `.devloop/<slug>/06-e2e.md`

```markdown
## E2E test artifacts

### Surface: gRPC | REST | GraphQL | CLI   (client: grpcurl | curl | binary)

### Generated calls
| # | Endpoint | Verifies |
|---|----------|----------|
| 1 | <name> | <what is verified> |

### Auth
- Requires <permission/role>: <list of endpoints>
- Open (no auth check): <list>

### How to run
Prerequisites: <client tool install>, full stack running, valid TOKEN.
```bash
ADDR=<addr> TOKEN=<token> bash .devloop/<slug>/e2e-test.sh
```

## Status: GENERATED — run manually against local stack or staging
```
